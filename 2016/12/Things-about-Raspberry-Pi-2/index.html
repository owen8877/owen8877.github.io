2016-12-09
pi

没事又去折腾树莓派……

这回是折腾 minecraft 服务器。众所周知， pi 性能孱弱，内存不足，自然需要跑一个优化过的版本。

按照小白指南，大概是这样：

  1. 从 这里 下载 BuildTools.jar 文件。
  2. java -jar BuildTools.jar
  3. 等，编译
  4. 编译完之后运行 spigot*.jar ，然后修改 eula.txt 同意最终用户许可协议，之后再运行服务器就可以跑了。

然鹅，并没有像↑说的那样一帆风顺。

首先是代理没配置好，第一步第二步直接跪。然后第三步我等了两个多钟头 那个说二十分钟的出来我保证不打死你 ，还有就是直到现在我还卡在第四步上。

完。

更新：树莓派3B+的性能还是太孱弱，比如划个船，服务器日志上就会写 ‘XXX moved too quickly!’ ，所以并不能跑起来，遂放弃。
