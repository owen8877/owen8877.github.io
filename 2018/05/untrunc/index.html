untrunc

2018-05-10
daily

起因：手机接 OTG 线录视频，结果空间太小没把尾部/头部的流信息写上，完全无法播放。
查到似乎 untrunc 可以尝试修复。

先尝试安装一些依赖；在 Arch 上似乎是 ffmpeg-full ：

？？？

还是先 clone 一下 repo 吧（
然后发现 / 没空间了 QAQ
清了一遍 cache ，发现 aur 上原来有现成的源；但试了一下似乎又不行，报错 std::bad_alloc ……

后来发现是采样用的视频太大了，装不进内存里。
那还是挺简单的，直接 ffmpeg 截一段就好。
